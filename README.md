Telecom Churn Analysis-Dashboard

Overview

This project demonstrates an end-to-end data analytics workflow using Python, SQL, and Power BI. The process includes loading and cleaning a dataset in Python, performing data analysis with SQL queries in MySQL Workbench, and creating an interactive Power BI dashboard for insights and visualization.
The goal of this project is to showcase practical data analytics skills including data preprocessing, querying, analysis, and dashboard development.

Dataset

The dataset used in this project contains structured data for analysis purposes. It was processed to handle missing values, remove duplicates, standardize formats, and prepare the data for reporting and visualization.
Dataset Features May Include:

Customer information

Sales records

Product details

Transaction data

Regional performance metrics

Tools & Technologies

Python -	Data loading and cleaning

Pandas -	Data manipulation and preprocessing

MySQL Workbench	- SQL querying and analysis

Power BI - Dashboard creation and visualization

Excel - Dataset source format

Project Workflow

1. Data Loading in Python
Imported dataset using Pandas

Explored rows, columns, and data types

Checked dataset structure and summary statistics

3. Data Cleaning
Removed duplicate records

Handled missing/null values

Corrected inconsistent data formats

Renamed columns for better readability

Prepared clean dataset for SQL analysis

5. SQL Analysis in MySQL Workbench

Performed SQL queries to extract business insights such as:

Total Customers who churned

Total revenue

Overall Churn Rate of customers

Customer trends

Aggregations using GROUP BY

Filtering using WHERE


4. Power BI Dashboard

Created an interactive dashboard to visualize key insights and performance metrics.

Dashboard Highlights:

KPI Cards

Churn by services

Category Performance

Interactive Filters & Slicers

Charts and Visual Reports
